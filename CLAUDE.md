# Ringer repo — routing rule

Swarm-shaped, model-calling, or repetitive work (probes, simulations,
harness/eval scripts, edit→test loops, batch edits across files) goes
through Ringer as a manifest, not inline. A single task is a one-task
manifest. Load the `ringer` skill before reaching for a script that calls
a model or drives an agent CLI directly.

See the `ringer` skill for the full playbook, patterns, and templates.
